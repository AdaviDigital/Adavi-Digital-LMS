# Code of Conduct

## Our Commitment

Adavi Digital Institute is committed to fostering a professional, respectful, inclusive, and collaborative environment for everyone who contributes to this project.

We believe that a positive community leads to better software, stronger collaboration, and continuous innovation.

By participating in this project, you agree to abide by this Code of Conduct.

---

# Our Standards

Examples of behavior that contribute to a positive environment include:

- Treat everyone with professionalism and respect.
- Be constructive when giving or receiving feedback.
- Encourage collaboration and knowledge sharing.
- Write clean, maintainable, and well-documented code.
- Respect differing viewpoints and experiences.
- Accept responsibility for mistakes and learn from them.
- Focus discussions on improving the project.

---

# Unacceptable Behavior

The following behaviors are unacceptable:

- Harassment of any kind.
- Personal attacks or insults.
- Discrimination based on race, gender, religion, nationality, disability, age, or other protected characteristics.
- Offensive language or inappropriate content.
- Intimidation or bullying.
- Publishing confidential information.
- Sharing passwords, API keys, or sensitive credentials.
- Deliberately introducing malicious code.
- Spamming discussions or pull requests.
- Any illegal or unethical activity.

---

# Coding Ethics

Contributors are expected to:

- Write secure code.
- Follow coding standards.
- Protect user privacy.
- Avoid plagiarism.
- Respect intellectual property rights.
- Use licensed third-party libraries appropriately.
- Keep project dependencies updated.

---

# Communication Guidelines

When communicating:

- Be respectful.
- Assume positive intent.
- Keep discussions technical and constructive.
- Provide evidence when suggesting improvements.
- Avoid emotional or offensive language.

Remember:

Critique code—not people.

---

# Pull Request Etiquette

Before submitting a Pull Request:

- Ensure the project builds successfully.
- Run all available tests.
- Update documentation if necessary.
- Remove debugging code.
- Follow the project's coding standards.
- Write meaningful commit messages.

---

# Issue Reporting

When reporting issues:

Include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser/Operating System
- Error messages

Please avoid duplicate reports.

---

# Security

Do not:

- Commit `.env` files.
- Commit API keys.
- Share passwords.
- Expose database credentials.
- Upload sensitive production data.

Security concerns should be reported privately to:

security@adavidigitalinstitute.com

---

# Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Violations may result in:

- A private warning
- Temporary suspension from contributing
- Removal of contributions
- Permanent restriction from the project

The appropriate action will depend on the nature and severity of the violation.

---

# Scope

This Code of Conduct applies to:

- Source code
- Documentation
- GitHub Issues
- Pull Requests
- Discussions
- Project communications
- Internal collaboration related to this repository
